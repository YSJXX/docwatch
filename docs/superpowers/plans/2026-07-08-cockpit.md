# docwatch v0.2 — Split Cockpit Implementation Plan

> Design of record: `~/.gstack/projects/YSJXX-docwatch/designs/dashboard-20260708/variant-G.html` (approved). Split cockpit: left dashboard rail + right doc-preview panel; click a rail item → that `.md` renders on the right without leaving the view.

**Goal:** Add a `/cockpit` two-pane view — left dashboard rail (Live Activity + recent commits + category rows, items clickable) + right panel rendering the selected document's markdown. Watch on the left, read on the right.

**Architecture:**
- **Bare preview route** `src/pages/preview/[...id].astro` — renders a single `docs` collection entry via Astro 5 `render()` into a minimal light-doc layout (no Starlight chrome), with client-side mermaid. This is the doc body the cockpit shows.
- **Cockpit page** `src/pages/cockpit.astro` — two-pane grid. Left rail reuses the dashboard's data (activity, commits, categories). Right pane is an `<iframe>` whose `src` is set to `/preview/<id>` on rail-item click. iframe (not innerHTML) so mermaid `<script>` runs and styles stay isolated.
- Keep the restyled `/dashboard` as-is (fallback). Flip `/` redirect from `/dashboard` to `/cockpit` in the final task once the cockpit is solid.

**Tech:** Astro 5.18, `astro:content` getEntry+render, existing scan/activity/category data layer, mermaid client, Playwright e2e.

## Global Constraints
- Reuse the existing data layer (`@/data/*`) and `DEFAULT_CONFIG`; do not re-implement scanning.
- Light GitHub theme tokens (match variant-G / the restyled dashboard): --canvas #f6f8fa, --panel #fff, --line #d0d7de, --line2 #eaeef2, --fg #1f2328, --blue #0969da, --green #1a7f37, --faint #8c959f.
- Do NOT break `/dashboard`, `/docs/*` (Starlight), `/api/activity.json`, or any test. Keep element ids used by e2e (#live-activity/#timeline/#cards on the dashboard).
- Entry `id` is the normalized collection id (e.g. `docs/adr/adr-001`). Use it verbatim for `/preview/<id>` and getEntry.
- Escape any dynamic text injected into HTML (reuse the esc() pattern from LiveActivityBanner).
- One commit per task, `feat:`/`test:` prefix. NEVER pkill (PID-file kills only).

### Task 1 — Bare document preview route
**Files:** Create `src/pages/preview/[...id].astro`, `src/components/DocPreview.astro` (optional inline), `src/components/Mermaid.astro` (reuse existing).
- `getStaticPaths` NOT usable (dynamic external content, SSR). Use `prerender = false` and read `Astro.params.id`.
- `const entry = await getEntry('docs', id)`; if missing → 404 response with a small "document not found" body.
- `const { Content } = await render(entry)`.
- Minimal layout: `<html><head>` with inline light-doc CSS (variant-G `.doc` styles: h1/h2/h3, p, ul, tables, code, task-lists) + `<body><article class="doc"><Content /></article>` + the existing client-side `<Mermaid />` component so ` ```mermaid ` blocks render.
- No Starlight sidebar/header/search — bare.
**Verify:** dev smoke against /home/ysj/Glance: `curl -s http://localhost:4321/preview/docs/adr/adr-001-mobile-framework-react-native-expo | grep -oE '<article|모바일 프레임워크|<h2'` shows the rendered ADR; confirm NO Starlight sidebar markup. A doc with a mermaid fence renders `<svg` after client JS (note: curl won't show SVG; just confirm the code block + Mermaid script present).
**Commit:** `feat: bare document preview route for cockpit`

### Task 2 — Cockpit page shell (two-pane) + preview iframe
**Files:** Create `src/pages/cockpit.astro`, `src/layouts/CockpitLayout.astro`.
- Full-height two-pane grid (left rail ~400px, right 1fr), header = the same LIVE strip as the restyled dashboard (LIVE, target repo, docs count, dirty count — SSR).
- Left rail: reuse the dashboard's SSR data — Live Activity (compact), recent commits (compact rows), category rows with the blue LED bars. Each item that maps to a doc gets `data-preview="/preview/<id>"` and role/tabindex for click.
- Right pane: `<iframe id="doc-frame" title="document preview">` + an initial empty-state overlay ("Select a document to preview").
- Client script: delegate click on `[data-preview]` → set `#doc-frame` src, remove empty state, set `.sel` highlight on the clicked item (clear others), and `history.replaceState` a `?doc=<id>` param. On load, if `?doc=` present, open it.
**Verify:** dev smoke: `/cockpit` 200; contains `id="doc-frame"`, the LIVE strip, and category rows; clicking is client-side (can't curl), so assert the markup + data-preview attrs exist.
**Commit:** `feat: cockpit two-pane shell with preview iframe`

### Task 3 — Rail item → document mapping + category doc lists
**Files:** Modify `src/pages/cockpit.astro`; maybe extend `src/data/paths.ts` (reverse map file path → entry id) with a tested helper.
- Recent commit rows / dirty files / recently-modified: map each file path to a docs entry id when one exists (a file is previewable only if it's in the collection). Compute the set of collection ids SSR; for each rail file, if it resolves to an id, make it clickable (`data-preview`), else render as plain text.
- Category rows: clicking a category expands/opens its document list (the docs whose resolved category === that category). Render the per-category doc list in the rail (collapsible) with each doc `data-preview`. Keep it simple: show the docs under each category as a small nested list.
- Unit test the path→id helper in `tests/unit/` (e.g. `docs/adr/ADR-001.md` → `docs/adr/adr-001`), covering dotted roots and non-doc files (returns null).
**Verify:** dev smoke: category rows list their docs with data-preview; a commit touching a doc file is clickable. Unit tests green.
**Commit:** `feat: map rail items to previewable documents + category doc lists`

### Task 4 — e2e + make cockpit the default
**Files:** Create `tests/e2e/cockpit.spec.ts`; modify `src/pages/index.astro`.

- Flip `src/pages/index.astro` redirect `/dashboard` → `/cockpit`. Keep `/dashboard` route working (fallback).
- Full gate: typecheck + unit + integration + e2e (LD_LIBRARY_PATH for chromium) all green.
