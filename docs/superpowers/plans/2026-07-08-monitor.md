# docwatch v0.3 — `/monitor` (Agent Docs Monitor, M3 3-column)

> Design of record: `docs/design/agent-docs-monitor/agent-docs-monitor.dc.html` (imported from claude.ai/design "AI 에이전트 파일 모니터링 대시보드"). Faithful visual port of the 3-column Material 3 dashboard, wired to docwatch's REAL data layer. New route — does NOT touch `/cockpit`, `/dashboard`, `/`, `/preview`.

**Goal:** Add `/monitor` — a Material 3 (dark-default + light toggle) 3-column live dashboard:
- Header: monitored project path · file stat chips (파일/NEW/MOD) · theme toggle · Live Tracking toggle.
- Col 1: 실시간 타임라인 (file events) + 프로젝트 파일 트리.
- Col 2: selected-file detail — NEW → meta chips + 요약 발췌; MOD → diff (stub in this phase).
- Col 3: rendered preview — plan file → checklist tracker; else markdown preview (existing `/preview` iframe).

## Scope decisions (locked)
- **New route only.** Keep `/cockpit` (v0.2) and `/dashboard` (v0.1) as-is. `/` still redirects to `/cockpit`.
- **Real data, SSR-embedded.** All per-doc data (status/size/lines/created/excerpt/plan items) computed at SSR and embedded as one JSON blob in the page. Client selection reads from it — **no new API endpoint** in this phase.
- **Live = polling** the EXISTING `/api/activity.json` (dirty + recentlyModified). Live toggle starts/stops the poll. This is the honest realization of "Live Tracking" (no fake event simulation like the mock).
- **Deferred (honest stubs / next phase):** git diff view (col2 MOD), AI summaries (use first-paragraph excerpt instead, labeled 발췌 not AI), persistent checkbox toggle write-back (toggle is local-only, non-persistent), server-pushed brand-new files (poll refresh only reflects dirty/mtime).

## Global constraints
- Reuse `@/data/*` + `DEFAULT_CONFIG`. Do NOT re-implement scanning. Use `getActivity`, `scanGitLog`, `scanDirtyFiles`(via activity), `scanCheckboxes`, `resolveCategory`, `aggregateByCategory`, `buildDocIndex`/`fileToDocId`, `targetRepoRoot`.
- M3 tokens: inline the imported `tokens/colors.css` + `tokens/typography.css` values as `is:global` CSS. Dark scheme is default (`data-theme="dark"` on the wrapper). Roles: `--md-sys-color-*`. Fonts: Roboto / Roboto Mono via Google Fonts `<link>` with system fallbacks (`system-ui`, `ui-monospace`) so it degrades if the browser can't fetch.
- Accent palette (NEW=green, MOD=orange) must be theme-aware — port the design's `palette()` light/dark values.
- Escape ALL dynamic text injected into HTML (reuse the `esc()` pattern). The SSR JSON blob must be emitted via `set:html={JSON.stringify(...)}` inside a `<script type="application/json">` (NOT interpolated into JS) to avoid injection.
- Selection wiring: reuse the proven cockpit pattern — click `[data-preview]` → set col3 `#doc-frame` iframe src to `/preview/<id>` (guard `startsWith('/preview/')`), highlight `.sel`, `history.replaceState('?doc=<id>')`. On load, honor `?doc=`.
- Do NOT break any existing route or test. One commit per task, `feat:`/`test:` prefix. NEVER pkill (PID-file kills only).

### Task 2 — `/monitor` page: SSR data + 3 columns + selection/live client
**Files:** Create `src/pages/monitor.astro`. `export const prerender = false`.
- **SSR data build** (mirror `cockpit.astro`'s data prelude):
  - `root`, `docs`, `activity = getActivity(root, DEFAULT_CONFIG)`, `commits = scanGitLog(root,20).catch(()=>[])`, `docIndex`/`sourceRelById`, `dirty = activity.dirty`.
  - `dirtyByRel: Map<sourceRelPath, 'NEW'|'MOD'>` — status `??`|`A`|`C` → NEW, `M`|`R` → MOD (ignore `D`).
  - `docsTotal = docs.length`; `newCount`/`modCount` = count of dirty entries whose path resolves to a doc, by mapped status (fallback: count all dirty by status if not a doc).
  - Per-doc detail map `details: Record<id, {name,dir,path,status,size,created,lines,excerpt,plan}>`:
    - read file at `join(root, sourceRelPath)` (catch→''); `lines` = line count; `size` = human bytes from `fs.stat`; `created` = mtime `MM-DD HH:mm`; `status` = dirtyByRel.get(rel) ?? 'MOD'|'—' (if clean, status 'DOC'/none); `excerpt` = first non-heading, non-empty paragraph (~280 chars); `plan` = `scanCheckboxes` gives counts, but ALSO parse per-item `{text, done}` list (new tiny parser: same fence/inline-code rules as `scanCheckboxes`, capture text after `[ ]`/`[x]`). `plan` present only if items.length>0.
  - **timeline**: merge `activity.recentlyModified` (has mtime → "N분 전") with `activity.dirty` resolved to docs; each item `{id?, name, dir, status, ago, preview}`; dedupe by rel, newest first, cap 16. Non-doc dirty files still listed (not clickable).
  - **tree**: build nested list from doc `sourceRelPath`s (dirs collapsible-static, files clickable). Node `{depth, isDir, name, id?, status?}`. Sort dirs-then-files, alpha.
- **Markup** in `MonitorLayout`:
  - Col1: 실시간 타임라인 section (timeline rows, `data-preview` when id) + 프로젝트 파일 트리 section (tree rows, `data-preview` when file+id).
  - Col2: `#detail-pane` with an empty-state ("타임라인·트리에서 파일 선택"); filled by client from `details`.
  - Col3: `#doc-frame` iframe + `#plan-pane` (hidden) + `#col3-empty` empty-state.
  - Emit `details` as `<script id="dw-details" type="application/json" set:html={JSON.stringify(details)}></script>`.
- **Client script:**
  - Parse `#dw-details` JSON once.
  - `select(id, el)`: highlight `.sel`; fill col2 `#detail-pane` from `details[id]` (NEW → meta chips + ✦ 문서 발췌 + excerpt; MOD → diff stub notice "라인 단위 diff는 다음 버전 — 아래는 문서 발췌" + excerpt); col3 → if `details[id].plan` render plan tracker into `#plan-pane` (show it, hide iframe), else set iframe `src=/preview/<id>` (show iframe, hide plan). `history.replaceState('?doc=/preview/<id>')`. All injected text via `textContent`/escaped, never raw innerHTML of file content.
  - Plan tracker: progress bar (done/total %), filter tabs 전체/완료/예정 (no 삭제됨 — plain md has no deleted concept; omit or show disabled 0), item rows with checkbox mark; clicking a checkbox toggles **local UI state only** (title tooltip notes "표시 전용 · 파일 미저장"). 
  - Delegated click + keyboard (Enter/Space) on `[data-preview]` (reuse cockpit handler).
  - On load: honor `?doc=` (find matching `[data-preview]` or set iframe directly).
  - **Live poll:** listen for `dw:live-toggle`. When live on: `setInterval` every 5s → `fetch('/api/activity.json')` → rebuild timeline rows + recompute NEW/MOD stat chips from returned dirty/recentlyModified; slide-in animation on new top row. When off: clear interval. Start according to initial `data-live`.
**Verify:** dev smoke against `/home/ysj/Glance`: `/monitor` 200; contains the 3 columns, `#doc-frame`, `#dw-details` JSON, timeline rows, tree rows. Clicking a doc row fills col2 + loads col3 (curl can't click — assert markup + `data-preview` attrs + the embedded JSON parses). Selecting a plan doc shows the tracker. `npm run typecheck` clean.
